# 计划文档审查者提示模板
在调度计划文档审查子代理时使用此模板。
**目的：** 验证计划完整、与规范匹配，并具有适当的任务分解。
**调度时机：** 完整计划编写后。
```
Task 工具（通用）：
  description: "审查计划文档"
  prompt: |
    你是计划文档审查者。验证此计划完整并准备好实现。
    **要审查的计划：** [PLAN_FILE_PATH]
    **参考规范：** [SPEC_FILE_PATH]
    ## 检查什么
    | 类别 | 要查找什么 |
    |------|----------|
    | 完整性 | TODO、占位符、不完整任务、缺失步骤 |
    | 规范对齐 | 计划覆盖规范需求，没有重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可执行 |
    | 可构建性 | 工程师能否遵循此计划而不卡住？ |
    ## 校准
    **只标记会在实现期间导致实际问题的问题。**
    实现者构建错误的东西或卡住是问题。
    措辞微小、风格偏好和"锦上添花"建议不是。
    除非有严重差距 — 规范中缺失的需求、矛盾的步骤、占位符内容或太模糊无法执行的任务，否则批准。
    ## 输出格式
    ## 计划审查
    **状态：** 已批准 | 发现问题
    **问题（如果有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为什么对实现重要]
    **建议（建议性，不阻止批准）：**
    - [改进建议]
```
**审查者返回：** 状态、问题（如果有）、建议
