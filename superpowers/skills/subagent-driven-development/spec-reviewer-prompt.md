# 规范合规性审查者提示模板

在调度规范合规性审查子代理时使用此模板。

**目的：** 验证实现者构建了请求的内容（不多不少）

```
Task 工具（通用）：
  description: "审查任务 N 的规范合规性"
  prompt: |
    你正在审查实现是否与其规范匹配。

    ## 请求了什么

    [任务需求的完整文本]

    ## 实现者声称构建了什么

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们实现的说法
    - 信任他们关于完整性的声称
    - 接受他们对需求的解释

    **要：**
    - 阅读他们编写的实际代码
    - 将实际实现与需求逐行比较
    - 检查他们声称实现但缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们实现了请求的所有内容吗？
    - 有他们跳过或遗漏的需求吗？
    - 他们声称某些东西有效但实际上没有实现吗？

    **额外/不需要的工作：**
    - 他们构建了未请求的内容吗？
    - 他们过度工程或添加了不必要的功能吗？
    - 他们添加了规范中没有的"锦上添花"吗？

    **误解：**
    - 他们对需求的解释与预期不同吗？
    - 他们解决了错误的问题吗？
    - 他们实现了正确的功能但方式错误吗？

    **通过阅读代码验证，而不是通过信任报告。**

    报告：
    - ✅ 规范合规（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外的内容，带有 file:line 引用]
```
