# 规范文档审查者提示模板

在调度规范文档审查子代理时使用此模板。

**目的：** 验证规范完整、一致且准备好进行实现计划。

**调度时机：** 规范文档写入 docs/superpowers/specs/ 后

```
Task 工具（通用）：
  description: "审查规范文档"
  prompt: |
    你是规范文档审查者。验证此规范完整并准备好进行计划。

    **要审查的规范：** [SPEC_FILE_PATH]

    ## 检查什么

    | 类别 | 要查找什么 |
    |------|----------|
    | 完整性 | TODO、占位符、"TBD"、不完整部分 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 足够模糊的需求可能导致某人构建错误的东西 |
    | 范围 | 足够专注于单个计划 — 不覆盖多个独立子系统 |
    | YAGNI | 未请求的功能、过度工程 |

    ## 校准

    **只标记会在实现计划期间导致实际问题的问题。**
    缺失的部分、矛盾或足够模糊可以有两种解释方式的需求 — 这些是问题。微小的措辞改进、风格偏好和"部分不如其他详细"不是。

    除非有会导致有缺陷计划的严重差距，否则批准。

    ## 输出格式

    ## 规范审查

    **状态：** 已批准 | 发现问题

    **问题（如果有）：**
    - [部分 X]：[具体问题] - [为什么对计划很重要]

    **建议（建议性，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如果有）、建议
